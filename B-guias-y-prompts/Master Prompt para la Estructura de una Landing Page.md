## Master Prompt para la Estructura de una Landing Page en WordPress (Divi 5)

### Bloque 0. Variables (a completar antes de la ejecución)
- {Tipo de Sitio}: [Corporativo, eCommerce, Servicio Local, Marca Personal, SaaS…]
- {Público Objetivo (TA)}: [p. ej. decisores B2B, propietarios de viviendas, estudiantes…]
- {Objetivo Principal de la Página}: [Captación de leads, ventas directas, reservas, suscripciones]
- {Tono y Estilo}: [Premium, Minimalista, Cercano, Corporativo…]
- {Región/Mercado}: [España, UK, Global]
- {CTA Principal}: [Reserva una cita / Comprar ahora / Solicitar presupuesto]
- {CTA Secundario}: [Suscribirse / Saber más / Descargar guía]

---

### Etapa 1. Tesis Estratégica
1. Definir la **Propuesta Única de Valor (UVP)**.  
2. Crear un **Perfil BDI** del público objetivo:  
   - **Creencias**: lo que ya piensan (a veces de forma errónea).  
   - **Deseos**: lo que realmente quieren conseguir.  
   - **Intenciones**: lo que deberían hacer justo después de ver la página.  
3. Principio rector: *Cada bloque debe generar confianza o conducir al CTA.*

---

### Etapa 2. Biblioteca de Bloques
*Bloques obligatorios (adaptar el orden según el tipo de sitio):*
1. **Hero** – titular, subtitular, CTA principal y visual fuerte.  
2. **Problema** – describir el dolor del cliente.  
3. **Solución / Oferta** – qué ofreces y cómo resuelve el problema.  
4. **Beneficios / Características** – 3–6 puntos claros con iconos.  
5. **Prueba Social** – testimonios, logotipos de clientes, cifras.  
6. **Cómo Funciona** – explicación en 3–4 pasos.  
7. **Portfolio / Caso de Éxito / Storytelling** – ejemplos o historia de un cliente.  
8. **Precios / Paquetes** – tabla comparativa clara.  
9. **Señales de Confianza** – garantías, certificaciones, partners.  
10. **FAQ** – 5–7 objeciones resueltas (con marcado schema JSON-LD).  
11. **CTA Final** – repetir la llamada a la acción con urgencia.  
12. **Footer** – navegación, contactos, legal.

*Bloques opcionales:*  
- Vídeo explicativo  
- Presentación del equipo  
- Blog / Recursos destacados  
- Calculadora o formulario interactivo  

---

### Etapa 3. Reglas de Secuenciación
- Empezar siempre con un **Hero potente**.  
- Alternar bloques **racionales** (características, proceso) con **emocionales** (testimonios, storytelling).  
- Incluir CTA al menos **3 veces** (Hero, mitad, final).  
- Colocar el FAQ **antes del CTA final**.  
- Cada 3er bloque debe neutralizar una **objeción típica** (precio, confianza, complejidad).  

---

### Etapa 4. Patrones UX & SEO
- El botón CTA debe estar **visible en el primer pantallazo** (desktop + móvil).  
- Legibilidad móvil: **≤6 líneas de texto por bloque, CTA en línea separada**.  
- Usar **H1 para el titular**, H2 para títulos de bloques, H3 para subapartados.  
- Añadir **FAQ schema** y, opcionalmente, **Review schema** en testimonios.  
- Formular títulos H2 como **preguntas semánticas** para captar long-tail.  
- Verificar el enlazado interno: botones/formularios → ancla o página correcta.  

---

### Etapa 5. Implementación en WordPress (Divi 5)
Para cada bloque, sugerir 1–2 módulos de Divi:
- **Hero** → Fullwidth Header + Button  
- **Beneficios** → Blurb con iconos  
- **Prueba Social** → Testimonial Slider / Logo Carousel  
- **Cómo Funciona** → Blurbs numerados  
- **Portfolio** → Gallery o Filterable Portfolio  
- **Precios** → Pricing Tables  
- **FAQ** → Toggle / Accordion (con JSON-LD en Code Module)  
- **CTA Final** → Call to Action con imagen de fondo  

---

### Etapa 6. Requisitos de Salida
- Entregar la estructura final como **lista numerada de bloques en orden lógico**.  
- Añadir una **justificación de 1–2 frases** por bloque.  
- Proponer **2–3 variantes de CTA** (principal + secundario).  
- Indicar **qué módulo de Divi** usar en cada bloque.  
- Resaltar **dónde añadir señales de confianza** (logos, badges, estadísticas).
